# PodJobs — Extracted Pod (Local Deployment)

This is a self-contained AI agent swarm generated by [PodJobs](https://podjobs.vercel.app). It runs 12 specialized agents locally using the Nous Research Hermes framework.

## Quick Start

### 1. Install Dependencies

```bash
# Install Hermes Agent SDK (from Nous Research)
pip install git+https://github.com/NousResearch/hermes-agent.git

# Install other requirements
pip install -r requirements.txt
```

Or use the auto-installer:
```bash
python install_hermes.py
```

### 2. Set an API Key

Hermes needs an LLM provider. Set one of these:

```bash
# Google Gemini (free tier)
export GEMINI_API_KEY="your-key"

# Or OpenRouter / OpenAI / Anthropic
export OPENROUTER_API_KEY="your-key"
```

### 3. Run the Swarm

**Hermes Mode** (recommended):
```bash
python run_hermes.py
```

**Multi-Engine Mode** (tries Hermes → Ollama → Static):
```bash
python run_swarm.py
```

### 4. View Results

After execution, the consensus report is saved to `swarm_report_local.md`.

## Files

| File | Purpose |
|---|---|
| `agents_config.json` | Full pod config — 12 agents with roles, specialties, and framework configs |
| `run_hermes.py` | Executes all 12 agents through the real Hermes AIAgent SDK |
| `run_swarm.py` | Multi-engine runner (Hermes → Ollama → Static fallback) |
| `install_hermes.py` | Auto-installs Hermes for your OS (Windows/Mac/Linux) |
| `requirements.txt` | Python dependencies |
| `swarm_report_local.md` | Generated after each run — Merkle-attested consensus report |

## Architecture

```
User Task → Agent 1 (Planner) → Agent 2 (Researcher) → ... → Agent 12 (Arbiter)
                                                                    ↓
                                                          Merkle Root Seal
                                                                    ↓
                                                        swarm_report_local.md
```

Each agent receives the previous agent's output, applies its specialty, and passes results forward. The final Consensus Arbiter synthesizes everything and seals the pipeline with a SHA-256 Merkle root.
